# Goal: 实现 Phase 3 主动巡视最小可用集

## One-sentence statement

让 agent 从"被动响应"升级为"主动巡视"——每天定时扫描仓库健康状态，发现异常时自动开 issue 提醒人类，信噪比 ≥ 3:1。

## Acceptance Criteria

- [ ] **巡视清单可配置**：`memory/watchlist.yml` 定义检查项，每项有 `id / enabled / threshold_days / description`。
- [ ] **扫描脚本可独立运行**：`scripts/scan_repo.py` 读取 watchlist，执行各检查，输出结构化 Markdown 报告；exit 0 无发现，exit 1 有发现。
- [ ] **至少 4 类默认检查**：stale-goals / missing-reflections / open-questions / audit-overrun。
- [ ] **主动巡视 workflow**：`.github/workflows/proactive-watch.yml` 每日 09:00 UTC 运行，调用 scan_repo.py，有发现时自动开 GitHub issue（label: `agent-watch`）。
- [ ] **proactive-scan skill**：`.agents/skills/proactive-scan/skill.md` 声明 cognitive_mode / collab_tier / model_tier，供 dispatcher 路由。
- [ ] **权限登记**：`policies/permissions.md` 加 proactive-watch.yml 行（issues: write, contents: read）。
- [ ] **测试覆盖**：`scripts/test_scan_repo.py` ≥ 15 unittest，全部通过。
- [ ] **反思笔记落档**：G-002 关闭时在 `reflections/` 追加 R-NNN.md 引用本 goal id。
