## Lessons learned

- `gh issue create --label <name>` 要求 label 在仓库中预先存在；建议在 workflow 中先用 `gh label create --force` 保证幂等，或在仓库设置中预建 `agent-watch` label。
- `--body-file` 比 `--body` 更安全，避免 stdout 含 shell 特殊字符时引发参数解析问题。R-007 已采用此方式。
- Phase 3 的两个 ticket（T-005 扫描引擎 + T-006 触发层）职责边界清晰，单 PR 范围约束有效防止越界。
