# Goal: 实现 Phase 2 认知架构最小可用集

## One-sentence statement

让本仓库的 agent 从"事件驱动的脚本调度器"升级为"具有持续目标 + 反思能力 + 工作集装配"的智能体,使 `docs/agent-cognitive-architecture.md` §6 / §7 / §9 三个核心机制可运行。

## Acceptance Criteria

- [ ] **Goal Stack 可读写**:存在 `scripts/goal_stack.py`,提供 `list / show / advance / set-status` 四个子命令,通过 CLI 即可对 `goals/G-NNN.md` 文件进行增量推进与状态切换。
- [ ] **反思闭环自动化**:`scripts/run_skill.py` 在每次 skill 调用结束后自动追加一条 `reflections/R-NNN.md`,内容包含四问回顾(目标是否达成 / 用了哪些记忆 / 缺什么 / 下次怎么改)。
- [ ] **Working Set 装配独立成步**:`scripts/run_skill.py` 调用一个新模块 `scripts/assemble_context.py` 来组装 ContextPack,而不是直接把整个 `MEMORY.md` 塞进 prompt。装配规则按 `docs/agent-cognitive-architecture.md` §7 的 Phase 2 朴素方案实现。

- [ ] **Skill 协议升级**:每个 `.agents/skills/<id>/skill.md` 新增 `cognitive_mode` 与 `collab_tier` 两个 front-matter 字段,`validate-skills.yml` 强制校验。
- [ ] **互联图同步**:`AGENTS.md` 与 `MEMORY.md` 的 Interconnection Map 包含 `goals/`、`reflections/`、`scripts/goal_stack.py`、`scripts/assemble_context.py` 的级联规则。
- [ ] **反思笔记落档**:G-001 关闭时,必须在 `reflections/` 下追加一条 `R-NNN.md` 引用本 goal id。

## Lessons learned

- Phase 1 的 `model_tier` 字段只区分模型贵贱,没区分思维方式。Phase 2 必须把"思维模式"独立出来(`cognitive_mode`),否则 `dispatcher.yml` 无法做对路由。
- AGENT-COLLAB.md 的 `Files in scope` 字段是防止 @Copilot 越界的关键护栏——架构文档与协议文件必须列入"禁改"。
- 当本地仓库与远程并发改动时,**未 push 的 Phase 工作可能被 pull 覆盖**。后续每次架构改动应:先 commit 立刻 push,不要积累多次未推送的改动;否则丢失重建成本高(本次重建即教训)。
